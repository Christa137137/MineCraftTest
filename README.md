# MineCraftTest

## 设计思想

我们项目提出一种大语言模型驱动的进化测试算法框架，为了验证框架有效性，选择Minecraft这一复杂开放世界游戏作为研究对象，选取3大物理规则作为实验数据集，把随机测试和传统突变测试作为测试基线。

本项目的核心思想是将 Minecraft 作为物理测试平台，通过自动化流程发现规则违背和行为异常。它将测试过程拆分为输入生成、环境执行、结果评估、数据筛选和迭代优化几个阶段。整个框架以模块化和可扩展性为原则，避免将所有逻辑集中在单一入口，而是让每个测试类型独立维护自己的生成、执行和评估策略。


## 项目说明

本项目是一个基于 Minecraft 的物理测试自动化框架。它通过测试用例生成、执行、评估、筛选、变异和重复检查，发现游戏内违反物理规则的情况。项目支持多种测试模式，适用于水力、光照、弹射等不同类型的物理交互场景。

## 目录结构

cli.py 是程序入口脚本，负责加载测试套件并启动主循环。src 目录按功能划分为 llm、rcon、test 和 utils 模块。

src/llm 提供大模型封装类 LLM，用于与外部对话接口交互。src/rcon 提供 MCRconClient，用于与 Minecraft 服务器进行远程命令通信。src/test 包含多个测试子模块，每个子模块负责对应测试类型的生成、执行、评估、变异、选择、筛选和重复检查。src/utils 提供日志记录和辅助工具。

## 入口与运行流程

cli.py 根据运行模式加载测试套件。load_test_suite 将模式映射到相应测试包，并加载执行、评估、变异、生成、检查、选择和过滤模块。

程序在初始化时会创建输出目录结构，包括 population、tasks 和 inconsistent_seeds 等路径。主循环由 run_loop 控制。在 random 模式下，它主要执行生成和测试。在其他模式下，它依次执行选择、变异、测试、评估和筛选。

run_loop 还负责时间限制控制、停滞轮次检测和条件性重新生成。若连续若干轮没有产生新不一致样本，系统会删除部分种子并重新生成输入，以避免迭代停滞。

## src/test 模块说明

test 目录包含以下子模块

- watertest
- waterrandomtest
- waterfuzztest
- lighttest
- lightrandomtest
- lightfuzztest
- projectiletest
- projectilerandomtest
- projectilefuzztest

每个子模块均包含测试用例定义、生成逻辑、执行逻辑、评估逻辑、变异逻辑、选择逻辑、筛选逻辑和重复检查逻辑。该设计保证了测试类型之间的可复用性和扩展性。

## 各子模块功能概述

watertest 是水相关物理场景的测试模块。它包含测试用例的输入格式、游戏世界中的场景重置、对象生成和动作执行。该模块加载对象目录和重力方块标识，通过行动序列驱动测试并对结果进行评估。它还负责管理方块和流体物理行为，确保水相关场景的生成与测试稳定。

waterrandomtest 是水测试的随机变体。它侧重于直接生成和执行测试用例，不包含选择和变异步骤。它仍然执行评估和重复检查，用于广泛探索水力行为空间。该模式适合快速发现常见异常，并可作为初始样本池的来源。

waterfuzztest 是水场景的模糊测试变体。它强调边界条件和异常输入的探索，通过大量候选用例寻找罕见的不一致情况。该模式通常用于发现细节错误和不稳定行为，适合对水力交互规则进行深度探测。

lighttest 是光照和相关物理交互的测试模块。它同样包含用例生成、执行和评估步骤，用于验证光照规则和方块行为是否符合预期。该模块关注光照传递、透明方块和光源交互等问题，并将结果作为异常检测的依据。

lightrandomtest 是光测试的随机模式。该模式省略选择和变异，依赖原始生成和执行过程来收集行为样本。它适合快速验证光照场景的多样性，使得测试覆盖更广而不是依赖局部优化。

lightfuzztest 是光照场景的模糊测试模式。它用于发现光照交互中的异常行为和边界情况。该模式更适合在光照规则复杂且交互多变时，捕获非线性问题和罕见错误。

projectiletest 是弹射体交互的测试套件。它针对飞行对象和投掷实体的动力学行为，定义测试用例并执行评估。该模块关注抛物线运动、碰撞反馈和实体受力等细节。

projectilerandomtest 是弹射体测试的随机模式。该模式用于广泛采样弹射行为，快速获得测试结果。它适合探索各类初始角度和速度组合，并作为多样性更高的样本生成方案。

projectilefuzztest 是弹射体模糊测试模式。它用于探测罕见的不一致和异常轨迹行为。该模式更适合在弹射物理规则复杂时，寻找边界行为与随机干扰导致的异常结果。

## 核心技术要点

框架遵循职责分离原则。执行模块负责与游戏服务器交互并驱动测试场景。评估模块负责分析结果、识别不一致样本。选择、过滤和变异模块负责进化式候选管理。重复检查模块避免重复样本和冗余评估。

MCRconClient 通过远程命令与 Minecraft 服务器通信，并支持命令执行、状态采集和连接管理。日志工具按时间记录消息，并写入文件以便后续分析。

## 输出目录结构

程序会为每次运行创建 session 输出目录，包含 population、tasks、inconsistent_seeds 等子目录。每次迭代会进一步创建 task 目录，用于存储选择、执行、变异、评估、测试用例和不一致样本数据，并保存时间统计信息。
